## 2. DC 프레임워크 핵심 문법

```html
<!-- 파일 구조 -->
<x-dc>
  <!-- 템플릿 HTML -->
</x-dc>
<script type="text/x-dc" data-dc-script>
class Component extends DCLogic {
  state = { ... };
  renderVals() { return { ...템플릿변수들 }; }
}
</script>
```

| 문법 | 설명 |
|------|------|
| `{{ 변수명 }}` | 값 출력 |
| `onClick="{{ 함수명 }}"` | 이벤트 바인딩 |

| `this.setState({...})` | 상태 업데이트 |
| `renderVals()` | 템플릿에 전달할 변수 객체 반환 |

### 4-2. State 구조

```javascript
state = {
  // 기존 탭 상태
  activeTab: 0,

  // 진단 모달 상태
  diagOpen: false,       // 모달 열림 여부
  diagStep: 0,           // 현재 단계 (0=닫힘, 1~5)
  diagType: null,        // 선택된 진단 유형 ('free'|'l2'|'l3'|'l4'|'b2b'|'kids')
  paymentPending: false, // 결제 단계 여부
  payMethod: 'card',     // 결제 수단

  // 페이지네이션
  qPerPage: 0,           // 페이지당 문항 수 (0=전체)
  curPage: 0,            // 현재 페이지

  // 사용자 정보
  dName: '',
  dPhone: '',
  dEmail: '',
  dTarget: '',           // 역할/대상 선택

  // 진단 진행
  selectedFields: [],    // 선택된 분야 배열
  curQ: 0,               // 현재 문항 번호
  answers: {},           // { 문항인덱스: 점수(0~4) }
  results: null,         // 계산된 결과 객체
}
```

### 4-3. 주요 메서드

| 메서드 | 역할 |
|--------|------|
| `qs()` | 현재 진단 유형의 문항 배열 반환 |
| `lbls()` | 현재 진단 유형의 응답 라벨 반환 |
| `calcResults()` | 축별 점수 계산 → 종합 점수 → 등급 산출 |
| `buildRadar(results)` | React SVG 레이더 차트 생성 (최대 8축) |
| `save(key, data)` | localStorage 저장 (진단 이력) |

### 4-4. 점수 계산 로직

```
각 문항 점수 = (선택값 / 4) × 100   (0~100점)
축별 점수 = 해당 축 문항들의 평균
종합 점수 = 전체 축 점수의 평균

등급 기준:
  A+ : 90점 이상
  A  : 80점 이상
  B  : 70점 이상
  C  : 60점 이상
  D  : 60점 미만
```

### 4-5. renderVals() 반환 변수 목록

**탭·카드 (기존)**
- `tabs` — 탭 배열 (label, onClick, 활성 스타일)
- `activeCards` — 현재 탭의 카드 배열

**진단 모달 열기**
- `openFreeDiag` — 무료진단 바로 시작

**모달 상태**
- `diagOpen`, `stepLabel`, `progressPct`
- `isStep1`, `isStep2`, `isPayStep`, `isStep3`, `isStep4`, `isStep5`

**Step 1 — 유형 선택**
- `diagTypeCards` — 6가지 진단 유형 카드 배열

**Step 2 — 정보 입력**
- `dName`, `dPhone`, `dEmail`
- `setDName`, `setDPhone`, `setDEmail`
- `targetOpts` — 역할/대상 선택 옵션 배열

**결제**
- `payMethods` — 결제 수단 배열 (카드/카카오/네이버)
- `selectedDiagLabel`, `diagPrice`
- `payF1`, `payF2`, `payF3`

**Step 3 — 분야 선택**
- `fieldOpts` — 8개 분야 선택 옵션 배열

**Step 4 — 문항 응답**
- `pageQItems` — 현재 페이지 문항 배열 (각 문항에 opts 포함)
- `qPerPageOpts` — 페이지당 문항 수 옵션
- `hasPrevPage`, `hasNextPage`, `isLastPage`
- `pageLabel`, `prevPage`, `nextPage`
- `answeredCount`, `allAnswered`, `notAllAnswered`
- `qAllPct`, `totalQ`

**Step 5 — 결과**
- `resultScore`, `resultGrade`, `gradeMsg`, `gradeColor`
- `radarChart` — React SVG 컴포넌트
- `topRisks`, `topStrengths` — 축별 점수 정렬 배열
- `isFreeResult`, `recoCourse`
- `upgradeCtas` — 업그레이드 버튼 배열

**버튼 상태**
- `nextBtnBg`, `nextBtnC` — Step1 다음 버튼 색상
- `s2Bg`, `s2C`, `s2Label` — Step2 다음 버튼
- `startBg`, `startC` — Step3 시작 버튼

**액션 함수**
- `closeDiag`, `prevStep`, `goStep2`
- `step2Next`, `cancelPayment`, `completePayment`
- `startQ`, `submitDiag`, `downloadResult`
